Por que este site existe

Objetivo e métricas de sucesso

O site não é vitrine genérica — é a ferramenta que sustenta o funil de vendas de 3 estágios (Fase 3) e os canais de marketing já definidos (Fase 4).

Job principal

Converter visitante (gestor de ISP, towerco ou integrador) em contato via WhatsApp — CTA único, repetido em cada seção, sem fricção de formulário longo.

Job secundário

Sustentar a credibilidade técnica via portfólio de dossiês de serviço (Fase 4) — provar competência antes mesmo do primeiro contato humano.

ADR-01 · Hospedagem e CMS
Contexto
Mediador já possui Hostinger contratada com cPanel/WordPress 1-clique. Pediu "CMS sem código" e "zero manutenção" simultaneamente.
Decisão
WordPress nativo na Hostinger resolve as duas exigências sem peça extra (Decap/Tina seriam redundantes).
Alternativas
Estático + CMS headless (mais peças móveis); Next.js/Vercel (over-engineering para o estágio atual).
Consequências
+ Usa o que já está pago · + CMS de verdade para o Mediador editar · − Superfície de manutenção de plugins/updates do WordPress (mitigado: poucos plugins).
ADR-02 · Tema sem page builder
Contexto
Risco de diluição de marca com temas "prontos"; Mediador prioriza performance e edita via Claude, não precisa de builder visual.
Decisão
GeneratePress ou Astra (versões gratuitas) + CSS customizado direto no tema, sem Elementor.
Alternativas
Elementor free (mais fácil editar sozinho, mas adiciona JS/CSS extra e risco de "visual de corretor de imóveis").
Consequências
+ Performance máxima, CSS 100% sob controle da marca · − Edição de layout futura passa pelo código (aceito: Mediador edita via Claude).
ADR-03 · Local de instalação durante construção
Contexto
Landing estática já ocupa o domínio raiz hoje; site completo precisa ser construído sem derrubar o que está no ar.
Decisão
WordPress instalado em subdomínio novo.rsainfralink.com.br; troca de apontamento no domínio raiz quando pronto.
Alternativas
Subpasta (/novo) — rejeitada: exigiria busca-e-troca de URLs no banco de dados na migração final.
Consequências
+ Migração final é só reapontamento de DNS/domínio, sem tocar no banco · − Subdomínio fica indexável temporariamente (mitigar com noindex até o lançamento).

Qualidade do produto

Requisitos não-funcionais

O público-alvo (gestor de ISP/towerco) frequentemente acessa via celular, possivelmente com conexão rural fraca. Performance não é luxo — é requisito de conversão.

⚡
Performance mobile-first

Poucos plugins, sem page builder pesado. Meta: Core Web Vitals "Bom" nas 3 métricas, mesmo em conexão 3G/4G fraca.

🎨
Fidelidade de marca

Tema Signal/NOC escuro herdado 1:1 da Enciclopédia — mesmas variáveis de cor, tipografia (Space Grotesk/Manrope/JetBrains Mono).

♿
Contraste WCAG AA

Mitigação do risco de legibilidade do fundo escuro (apontado pelo Brand Chief): contraste mínimo AA em todo texto, tipografia maior que o padrão em mobile.

📱
Responsivo real

Técnico de campo acessa pelo celular. Testar em viewport real de 360-390px, não só redimensionar desktop.

💬
WhatsApp click-to-chat

Todo CTA usa wa.me com mensagem pré-preenchida contextual (varia por página/serviço). Botão flutuante presente em todas as páginas.

🔍
SEO técnico básico

Title/meta description por página com termos regionais (Fase 4): "serviço de campo telecom RS SC PR", "manutenção de torres SC" etc. Sitemap XML + Search Console.

🚫
Restrição de naming

Regra inegociável da Fase 1: nenhuma menção a "Engenharia" em qualquer copy, meta tag, alt text ou nome de arquivo.

🔒
Segurança básica

WordPress + plugins sempre atualizados; subdomínio de construção com noindex até o lançamento; HTTPS obrigatório (SSL gratuito via Hostinger).

Plano de execução

Checklist de implementação faseada

A landing estática (já entregue) já está no ar/pronta para subir. O que segue é a construção do site completo em paralelo, sem pressa.

Etapa 1
Infraestrutura
  • Criar subdomínio novo.rsainfralink.com.br via hPanel
  • Instalar WordPress 1-clique nesse subdomínio
  • Instalar tema GeneratePress ou Astra (decisão final no início desta etapa)
  • Configurar noindex temporário (não indexar o subdomínio de construção)
Etapa 2
Tema e identidade
  • Aplicar CSS customizado com as variáveis de marca (paleta, tipografia Google Fonts)
  • Subir logos vetorizadas (entregues nesta fase) como assets do tema
  • Configurar favicon a partir do símbolo isolado
Etapa 3
Páginas estáticas
  • Codificar Home, Serviços, Sobre, Contato direto no tema (templates customizados)
  • Configurar categoria "Portfólio" e template de listagem/single post
  • WhatsApp click-to-chat com mensagens contextuais por página/serviço
Etapa 4
Conteúdo e QA
  • Publicar primeiros dossiês reais (assim que Rafael enviar fotos) na categoria Portfólio
  • QA de performance (Core Web Vitals), responsivo real, contraste
  • SEO básico: title/meta por página, sitemap, Search Console
Etapa 5
Lançamento
  • Remover noindex do site completo
  • Trocar apontamento do domínio raiz (rsainfralink.com.br) do estático para o WordPress
  • Validar que a landing estática sai de circulação sem quebrar links já compartilhados
Não bloqueante: a escolha final entre GeneratePress e Astra pode ocorrer no início da Etapa 1 de implementação — ambos pré-aprovados pelo Conselho, sem necessidade de nova rodada de decisão.
